# G2/G3 Arc Support Implementation for GCodeZAA

## Summary

This document describes the implementation of proper G2/G3 arc command support in the GCodeZAA post-processor for the Bambulab X1C with X1Plus firmware.

## Overview

**Status:** ✅ Implemented and tested

The GCodeZAA repository (originally from GitHub) has been modified to support G2 and G3 arc movement commands. These commands are crucial for reducing file size and improving print quality by replacing multiple line segments with smooth arc movements.

### What Changed

**File: `GCodeZAA/gcodezaa/extrusion.py`**
- Added `decompose_arc()` function for converting arc commands into line segment waypoints
- Supports both I/J center offset and R radius parameter formats
- Handles clockwise (G2) and counter-clockwise (G3) directions
- Supports helical interpolation (Z-axis movement during arc)

**File: `GCodeZAA/gcodezaa/process.py`**
- Replaced G2/G3 TODO stubs with full implementations
- Parses G2/G3 parameters: X, Y, Z, I, J, R, E, F
- Decomposes arcs into line segments for ZAA surface analysis
- Maintains proper position tracking and extrusion distribution

## Technical Details

### Arc Parameter Support

#### G2 (Clockwise Arc)
```
G2 X<end_x> Y<end_y> [I<offset_i>] [J<offset_j>] [R<radius>] [Z<end_z>] [E<extrusion>] [F<feedrate>]
```

#### G3 (Counter-clockwise Arc)
```
G3 X<end_x> Y<end_y> [I<offset_i>] [J<offset_j>] [R<radius>] [Z<end_z>] [E<extrusion>] [F<feedrate>]
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| X, Y | float | End position coordinates (absolute or relative) |
| Z | float | End Z position (absolute) |
| I, J | float | Center offset from current position (XY plane) |
| R | float | Arc radius (alternative to I/J) |
| E | float | Extrusion amount |
| F | float | Feedrate |

### Decomposition Algorithm

1. **Center Calculation**
   - If I/J provided: Center = Current Position + (I, J)
   - If R provided: Calculate center using radius and chord length

2. **Angle Calculation**
   - Calculate angle from current position to center
   - Calculate angle from end position to center
   - Adjust for arc direction (clockwise vs counter-clockwise)

3. **Segment Generation**
   - Arc length = radius × angle_difference
   - Number of segments = ceil(arc_length / segment_length)
   - Default segment length: 1.0 mm (configurable)
   - Generate waypoints along arc using parametric equations

4. **Extrusion Distribution**
   - Distribute total extrusion proportionally across segments
   - Maintains output consistency with input extrusion

## Usage Example

### Input G-code (from Bambu Studio)
```gcode
; Generated by BambuStudio
G28
G92 X0 Y0 Z0 E0
G1 X10 Y0 Z0.2 F3000
G2 X20 Y10 I10 J0 E5 F1200    ; Clockwise quarter circle
G3 X30 Y10 I5 J0 E5 F1200     ; Counter-clockwise quarter circle
G0 X0 Y0
```

### Processing
```python
from gcodezaa.process import process_gcode

gcode = read_gcode("input.gcode")
output = process_gcode(gcode, "model_directory")
save_gcode(output, "output.gcode")
```

### Output
- Arc commands are decomposed into line segments for ZAA analysis
- Surface analysis (if STL models available) applies to arc segments
- Arc precision maintained within tolerance
- Extrusion values properly distributed

## Known Limitations

1. **Arc Decomposition:** Arcs are decomposed into line segments for ZAA analysis
   - Achieves high fidelity with 1mm segments
   - Can be tuned via `segment_length` parameter

2. **Relative Position Arcs:** Assumes arc parameters in absolute coordinates
   - I/J are always relative to current position
   - X/Y end positions converted to absolute internally

3. **Full Circles:** Special handling for 360° arcs
   - End position must equal start position
   - Function correctly handles edge case
