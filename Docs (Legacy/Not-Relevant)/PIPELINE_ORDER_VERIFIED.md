## Executive Summary

The pipeline order in `Ultra_Optimizer.py` has been verified as **CORRECT** and is optimized for your workflow where **OrcaSlicer arc fitting is disabled**.

**Pipeline Order:**
```
OrcaSlicer (G1 only) → Stage 1: Kinematic Optimizer → Stage 2: ArcWelder → Stage 3: GCodeZAA
```

---

## Why This Order is Optimal

### Stage 1: Kinematic Optimization (in process_gcode)

**Input:** Pure G1 commands from OrcaSlicer (no arcs)  
**Process:**
1. Analyzes angle between consecutive G1 segments
2. Calculates optimal acceleration profiles using kinematic analysis
3. Injects M204 commands to control nozzle acceleration
4. ZAA framework enabled for surface analysis eligibility
5. Arc parsing support for handling any G2/G3 that might exist
6. **Outputs:** G1 segments with optimized M204 acceleration commands

**Why First:**
- ✅ Works on raw G1 data before compression
- ✅ No information loss - analyzes actual turn angles
- ✅ M204 commands injected at optimal points
- ✅ Can detect patterns that ArcWelder will compress

### Stage 2: ArcWelder (integrated in process_gcode)

**Input:** Optimized G1 + M204 commands from Stage 1  
**Process:**
1. Reads Stage 1 optimized output
2. Analyzes G1 segment sequences
3. Compresses G1 sequences into G2/G3 arcs where beneficial
4. **Preserves all M204 commands** between arcs
5. Uses your configured parameters:
   - Dynamic resolution (`-d`)
   - Z-axis support (`-z`)
   - Y-axis support (`-y`)
   - Tolerance: 0.10 (10%)
   - Max error: 0.06mm

**Why Second:**
- ✅ Takes kinematically-optimized G1 and compresses it
- ✅ M204 commands still apply to arcs
- ✅ Arc fitting happens after motion analysis
- ✅ More efficient compression with pre-optimized data

### Stage 3: GCodeZAA (if models available)

**Input:** Arc-compressed G-code from Stages 1+2  
**Process:**
1. If STL models found in `stl_models/`:
   - Advanced surface raycasting on final output
   - Z-axis compensation based on surface normals
   - Applies to both G1 and G2/G3 arcs
2. If no models:
   - Stage skipped (optional)

**Why Third/Last:**
- ✅ Works on final optimized + compressed motion
- ✅ Can analyze actual movement topology
- ✅ Raycasting on simplified (arc) paths for efficiency
- ✅ Non-destructive enhancement

## Why OrcaSlicer Arc Fitting Must Be OFF

**Your Configuration (Arc Fitting = OFF):**
- ✅ OrcaSlicer outputs ONLY G1 commands
- ✅ Allows Stage 1 to see and analyze all turn angles
- ✅ ArcWelder compression is more efficient than OrcaSlicer's
- ✅ Pipeline optimization works perfectly

**If Arc Fitting Were ON:**
- ❌ OrcaSlicer would output G1 + G2/G3 (mixed)
- ❌ Stage 1 kinematic analyzer couldn't analyze arc-to-G1 transitions
- ❌ M204 placement could be suboptimal
- ❌ Defeats purpose of carefully-tuned pipeline

**Decision:** Keep arc fitting **DISABLED** in OrcaSlicer ✅
